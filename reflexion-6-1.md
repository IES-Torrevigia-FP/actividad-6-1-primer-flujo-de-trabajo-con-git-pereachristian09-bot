1. Explica con tus palabras la diferencia entre:

    Directorio de trabajo.
    Zona de staging.
    Historial de commits.

La diferencia que hay son que el directorio de trabajo es el repositorio donde estamos trabajando ya sea creando archivos, modificandolos ect, mientras que la Zona de staging esperan los archivos para ser añadidos al commit y el historial de commit sirve para ver que commits han sido añadidos o modificados al repositorio.



2. Describe un error típico que podría cometer alguien que empieza con Git en este flujo (por ejemplo, olvidarse de git add) y cómo lo evitarías.

Un error tipicio aparte de olvidarse de git add creo que seria confiarse como tal, y no hacer git status para comprobar que el archivo esta en untracked o en staging, creo que lo puede arreglar intentando poner git status cada que se acuerde o cada que pueda para que se acostumbre



3. ¿Qué ventaja tiene poder decidir qué cambios entran en cada commit (gracias a la zona de staging)?


Las ventajas que tiene poder decidir que cambios entran en cada commit es que es mas facil encontrar errores ya que al decidir cambios puedes añadir tanto poquitos cambios o grandes y es mas facil encontrarlos porque por ejemplo si yo tengo que añadir 3 commits, con el primero todo funcionaba perfecto y añado el segundo y deja de funcionar sabemos que el fallo se encuentra en el commit 2 ademas de que al poder decidir que cambio entra en cada commit puedes trabajar rapido.
